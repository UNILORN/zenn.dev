Terraform の AWS Provider では `provider "aws"` の `profile` にローカルの AWS Profile を指定できます。

https://registry.terraform.io/providers/hashicorp/aws/latest/docs

例えば以下です。

```tf
provider "aws" {
  region  = var.aws_region
  profile = var.aws_profile
}

variable "aws_region" {
  type    = string
  default = "ap-northeast-1"
}

variable "aws_profile" {
  type = string
}
```

`profile` を変数にしておくと、`terraform apply` 時に Profile 名を切り替えられます。

```sh
terraform apply -var='aws_profile=dev'
terraform apply -var='aws_profile=prd'
```

`tfvars` をローカルに置いておく形でも大丈夫です。

```tf
aws_profile = "dev"
aws_region  = "ap-northeast-1"
```

```sh
terraform apply -var-file=local.tfvars
```

この `local.tfvars` を `.gitignore` しておくと、各自のローカル Profile 名をコミットせずに運用できるので楽です。

ローカルの Profile 名が人によって違う場合でも、`-var` で渡せば同じ Terraform のコードを使い回せます。

例えば以下のような差分があっても対応できます。

- 自分は `prd`
- 別のメンバーは `production`

固定値で `profile = "prd"` のように書くよりは、変数化しておく方が扱いやすいです。
