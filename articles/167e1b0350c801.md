## はじめに

コミットメッセージ、毎回ちゃんと書こうとは思うのですが、変更内容を頭の中で整理して短くまとめるのは意外と面倒です。

特に「実装自体は終わったけど、最後にコミットメッセージで少し止まる」という時間が地味に積み重なるのが気になっていたので、作ったのが以下のツールです。

https://github.com/UNILORN/generative-commit-message-for-ai-tool

`generative-commit-message-for-ai-tool` は、**Git のステージング済み変更を見て、AI がコミットメッセージを生成してくれる CLI ツール**です。

紹介記事なので細かい実装の話は省きますが、日々の開発フローにかなり自然に入り込むツールになったので、今回はその使い方と運用 Tips をまとめます。

ちなみに、昨今の `claude code` や `codex` のように、エージェントの内部フローでそのまま `git commit` まで面倒を見てくれる使い方をしている人にとっては必要ないかなと思います。
skillsとかにしておけば不要だと思いますので。

普段のターミナル作業の中で「gitの操作させたくない」と感じる人には、ちょうどよくハマるかなと。。

## 使い方

詳しくはREADMEみてください。

- AWS Bedrock
- Claude API
- Claude Code
- Gemini CLI
- Copilot CLI
- Codex CLI

上記のCLIであればどのツールでも似たようなコミットメッセージを作成できます。

そのままだとコマンド名が長いので、私は `gcm` にリネームして使っています。

```bash
go install github.com/UNILORN/generative-commit-message-for-ai-tool@latest
mv $(go env GOPATH)/bin/generative-commit-message-for-ai-tool $(go env GOPATH)/bin/gcm
```

あとはいつも通り変更をステージして、コマンドを実行するだけです。

```bash
git add .
gcm
```

利用可能な環境があれば自動検出でも動きますし、`--provider` と `--model` を指定して明示的に使い分けることもできます。

例えば Claude Code を使うなら、こんな感じです。

```bash
gcm --provider=claudecode --model="claude-sonnet-4.5"
```

Gemini CLI や Copilot CLI、Codex CLI などを切り替えながら試せるのがこのツールの良いところです。

## alias紹介

私は Fish Shell を使っているので、よく使うプロバイダーは alias を切っています。

```fish
alias gc-claude='gcm --provider=claudecode --model="haiku-4.5"'
alias gc-copilot='gcm --provider=copilotcli --model="gpt-5.3-codex"'
alias gc-gemini='gcm --provider=geminicli'
alias gc-codex='gcm --provider=codexcli --model="gpt-5.3-codex-spark"'
```

## コミットもワンコマンドで

さらに一歩進めると、生成結果のタイトル行だけを使ってそのまま `git commit` する alias も作れます。

```fish
alias gca='git commit -m "$(gc-claude | gsed -z "s/---.*//g")";git log -n 1;git log --oneline -n 1'
alias gca-copilot='git commit -m "$(gc-copilot | gsed -z "s/---.*//g")";git log -n 1;git log --oneline -n 1'
alias gca-gemini='git commit -m "$(gc-gemini | gsed -z "s/---.*//g")";git log -n 1;git log --oneline -n 1'
alias gca-codex='git commit -m "$(gc-codex | gsed -z "s/---.*//g")";git log -n 1;git log --oneline -n 1'
```

この alias は、ツールの出力のうち `---` より前だけをコミットメッセージとして使っています。

特に小さめの修正や、明確に1コミットでまとまっている変更では、ほぼこの運用で困りません。

## もし生成結果が微妙だったら

私の場合はほとんどそういったことが無いので、
いったん直前のコミットだけ取り消して、好きなメッセージでコミットし直しています。

```bash
git reset --soft HEAD^
```

`gca-codex` のような一発コミット alias と、この `git reset --soft HEAD^` の二つくらいでおおよそ事足りてます。
